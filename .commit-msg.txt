fix(ai): correct Sonnet model name + friendlier error UX + README/DECISIONS docs

The Need Assistance? AI chatbot was non-functional for two reasons:

1. app/api/ai/chat/route.ts called Anthropic with model "claude-sonnet-4-6", which does not exist in Anthropic's model registry. Every chat request returned model_not_found_error even if the API key was set. Fixed to "claude-sonnet-4-5" (current Sonnet alias).

2. ANTHROPIC_API_KEY was not set on production Vercel, so the chatbot returned a 503 with raw JSON ({"error":"AI assistant is not configured..."}) that leaked through to end-user chat bubbles. Replaced the catch block in components/dashboard/ai-assistant.tsx with status-aware friendly messages: 503 -> "AI assistant is temporarily unavailable", 429 -> "today's limit reached", 401/403 -> "no access on this account".

3. README labeled ANTHROPIC_API_KEY as Optional but the feature has no graceful fallback - the chatbot strictly needs it. Moved to a dedicated AI Assistant section with the console.anthropic.com link and the Vercel-env-add steps so future operators can wire it up without re-discovering the failure mode.

4. DECISIONS.md D76 records the model-name fix and the key-required clarification so the audit trail catches the bad D5 entry that originally introduced the fictitious model name.

After this merges, the only remaining step is setting ANTHROPIC_API_KEY on Vercel (Production target) and triggering a redeploy. Once the env var is live, asking "What is GST?" in the chatbot returns a streaming Claude response.
